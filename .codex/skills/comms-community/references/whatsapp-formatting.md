# WhatsApp Formatting Rules

Use when drafting WhatsApp, SMS-style, or mobile-chat copy.

## Contract

1. Use `*bold*` for the main anchor phrase or decision.
2. Use `_italic_` sparingly for qualifiers such as `_not shipped yet_`.
3. Use `~strikethrough~` only for rejected wording the user should not send.
4. Use triple backticks only for exact copy/paste snippets.
5. Do not use Markdown headings, tables, masked links, blockquotes, nested bullets, or Discord-only syntax.
6. Keep paragraphs short. One idea per paragraph.
7. Prefer hyphen bullets for KPI lists.
8. Keep default output under roughly one phone screen unless the user asks for a longer note.
9. Put the WhatsApp-ready body in a fenced `text` block so the user can copy it cleanly.
10. If links are needed, paste the raw URL on its own line; otherwise omit links.

## Pattern

```text
*<main decision or framing>*

<one or two short sentences>

*KPIs / gates*
- <point>
- <point>
- <point>

<optional closing line>
```

For corrective roadmap replies:

```text
*Short version:* <direct correction>

<one sentence with the safer framing.>

*Non-negotiables*
- <trust boundary>
- <owner>
- <test/acceptance gate>
```
